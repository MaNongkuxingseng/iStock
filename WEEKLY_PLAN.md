### 技术决策记录

#### 1. 技术栈选择确认
- **后端框架**: FastAPI（异步支持好，自动文档生成）
- **前端框架**: React + TypeScript + Vite（现代前端生态）
- **数据库**: PostgreSQL（关系型，功能丰富）
- **缓存**: Redis（高性能，支持多种数据结构）
- **容器化**: Docker + Docker Compose（标准化部署）

#### 2. 开发规范制定
- **代码风格**: Black + isort（自动格式化）
- **代码检查**: flake8 + mypy（静态分析）
- **提交规范**: Conventional Commits（标准化提交信息）
- **分支策略**: Git Flow（功能分支+发布分支）

#### 3. 部署策略确定
- **开发环境**: Docker Compose本地运行
- **测试环境**: GitHub Actions自动化部署
- **生产环境**: 云服务器 + Docker部署
- **监控方案**: Prometheus + Grafana

### 常见问题解决

#### 环境问题
```bash
# Docker启动失败
docker-compose down -v
docker-compose up -d --build

# 数据库连接问题
docker-compose logs postgres
docker-compose exec postgres psql -U mystock_user -d mystock_ai

# 依赖安装问题
rm -rf venv
python -m venv venv
source venv/bin/activate
pip install -r requirements.txt
```

#### Git问题
```bash
# 分支同步
git fetch origin
git rebase origin/main

# 解决冲突
git mergetool
git add .
git rebase --continue

# 撤销更改
git reset --hard HEAD
git clean -fd
```

#### 开发问题
```bash
# 代码格式化
black src/
isort src/

# 代码检查
flake8 src/
mypy src/

# 运行测试
pytest tests/ -v
```
